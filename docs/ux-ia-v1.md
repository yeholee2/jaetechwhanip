## 1. 정체성 (Identity) — 한 문장

> **"한국 사람을 위한 ETF 한입 사전 + 진단."**
> 
> 토스보다 깊고, FunETF보다 친절하게 — 국내·미국 ETF를 한 화면에서
> 이해하고 비교하고 내 포트폴리오에 더한다.

### Why ETF한입?

| | 토스 주식정보 | 네이버 금융 | FunETF | **ETF한입** |
|---|---|---|---|---|
| 진입장벽 | 매우 낮음 | 중간 | 매우 높음 | **낮음** (한입 톤) |
| ETF 전문성 | 얕음 | 얕음 | 매우 깊음 | **중간** (한입 깊이) |
| 국내+미국 통합 | △ | △ | ✗ | **✓** |
| 친절한 한줄 요약 | ✓ (AI) | ✗ | ✗ | **✓** (룰베이스, 향후 AI) |
| 포트폴리오 진단 | ✗ | ✗ | ✗ | **✓** (한입 차별점) |
| 커뮤니티·콘텐츠 | △ | △ | ✗ | **✓** (스파링) |

### 페르소나 — "ETF 새내기 직장인"

- 30대, 월급쟁이, 자산 5천만 이하
- ETF는 들어봤지만 KODEX와 TIGER 차이 모름
- 토스는 너무 가볍고 키움은 너무 무서움
- **알고 싶은 것**: 이거 뭐 추종해? / 안전해? / 나한테 맞아?
- **결정에 필요한 5가지**: ① 무엇을 추종 ② 비용 ③ 분배 ④ 운용규모 ⑤ 위험

## 2. 핵심 5가지 정보 (ETF 상세 페이지에서 최우선)

**모든 ETF 상세 페이지 첫 화면에 이 5개가 한 눈에 보여야 함.**

1. **무엇을 추종하나** — 기초지수 + 카테고리 + 추종국가
2. **비용** — 총보수 (동종 평균 대비)
3. **분배** — 주기 + 최근 수익률
4. **운용규모** — AUM + 운용사
5. **위험** — 1~5 등급 + 환노출 + 변동성

### Toss·FunETF에서 차용할 만한 것 (우선순위 순)

| 정보 | 데이터 |
|---|---|
| **추종 지수 정보** | DB `bssIdxIdxNm` 이미 있음 — 노출만 필요 |
| **유사 ETF 추천** | "이거 보는 사람들이 본 ETF" — 카테고리+테마 매칭 |
| **운용사 정보 카드** | 운용사별 ETF 갯수, 대표 ETF |
| **변동성 (1Y σ)** | etf_daily_prices 채우면 계산 가능 |
| **MDD (최대낙폭)** | etf_daily_prices 채우면 계산 가능 |
| **추적오차** | KRX 공식 자료 별도 필요 (후순위) |
| **백테스트** | etf_daily_prices 풀로 계산 (후순위) |

## 3. IA (Information Architecture)

### 현재 — 평평하고 흩어짐

```
홈 / ETF / MY포트폴리오 / 피드 / 스파링
+ /q, /questions, /columns, /articles, /topics, /sparring (콘텐츠 4 채널)
+ /etf/themes, /etf/news, /etf/compare, /etf/all (ETF 하위 4개)
```

**문제:**
- "스파링"·"피드"·"칼럼"·"질문" 차이가 사용자에게 안 보임
- 콘텐츠 4 채널이 분산
- ETF 하위 페이지 위계 흐림
- 27개 라우트 = 결정 마비

### 제안 — **4개 메인 + 사라지는 서브**

```
┌─────────────────────────────────────────┐
│ 1. 홈            — 오늘의 한입            │
│ 2. ETF           — 발견 / 검색 / 상세     │
│ 3. MY            — 포트폴리오 진단         │
│ 4. 배우기         — 콘텐츠 통합             │
└─────────────────────────────────────────┘
```

#### 1. 홈 (`/`) — "오늘의 한입"
- 시장 한 줄 ("코스피 +0.5%, 나스닥 -1.2%")
- 추천 ETF 3개 (코어/배당/성장 — 페르소나 맞춤)
- 진행 중 학습 콘텐츠 1개
- 비로그인 → 회원가입 CTA

#### 2. ETF (`/etf`, `/etf/all`, `/etf/[slug]`)
- `/etf` = 발견 (랭킹 + 테마 + 큐레이션)
- `/etf/all` = 검색·필터 (국내/미국 통합)
- `/etf/[slug]` = 상세 (5단 구조 — 아래)
- `/etf/compare` 유지 (비교는 강한 기능)
- ~~`/etf/themes`, `/etf/news`~~ → `/etf` 안의 섹션으로 흡수

#### 3. MY (`/portfolio`)
- 진단 카드 (이미 있음)
- 보유 종목 / 운용사 분산
- 추천 액션 (다음 단계)

#### 4. 배우기 (`/learn`) — **신규 통합 라우트**
- `/q/*`, `/questions/*`, `/articles/*`, `/columns/*`, `/feed/*` → `/learn` 1개로 흡수
- 탭: ETF 기초 / 칼럼 / 뉴스 / Q&A
- 검색 + 카테고리 필터
- ~~`/topics/[slug]`~~ → `/learn?topic=` 쿼리로
- ~~`/sparring/[slug]`~~ → 커뮤니티로 분리 또는 `/learn?type=debate`

#### (옵션) 커뮤니티 (`/community`) — 후순위
- 스파링·투표·댓글
- 지금은 `/learn` 하위 탭으로 넣고, 트래픽 보고 분리

### GNB 최종

```
홈      ETF      MY      배우기      (검색 🔍)
```

5개 → **4개 + 검색 icon**. 모바일 bottomNav도 동일.

### Cleanup 대상

- 삭제 또는 통합: `/temp-etf`, `/temp-etf-toss`, `/temp-etf-v2`, `/temp-home`
- 통합: `/q/*` + `/questions/*` → 하나로 (둘 다 질문이라 혼란)
- 통합: `/articles/*` + `/columns/*` + `/feed/*` → `/learn/*`
- 유지: `/sparring/*` (당분간, 후속 통합 검토)
- 관리자: `/admin/*` 유지

## 4. ETF 상세 페이지 — 5단 통일 구조

토스·FunETF 둘 다에서 통하는 **세로 흐름**.

```
┌─ HERO ────────────────────────────────────┐
│  ETF 이름 + 코드 + 🇺🇸/🇰🇷 시장 Badge      │
│  운용사 · 카테고리                            │
│  한 줄 요약 (한입 한줄평)                     │
│  [관심][비교][매수가이드][AI에게 묻기]         │
└──────────────────────────────────────────┘

┌─ 핵심 5 ─────────────────────────────────┐
│  현재가 │ 추종 │ 보수 │ 분배 │ 위험         │
│   숫자  │  지수 │ 동종 │  주기 │  1~5      │
└──────────────────────────────────────────┘

┌─ 차트 (가격 + 분배) ────────────────────────┐
│  1M / 6M / 1Y / 5Y 토글                    │
└──────────────────────────────────────────┘

┌─ 구성 (Composition) ─────────────────────┐
│  섹터 도넛 + 상위 10 종목 + 추종 지수       │
└──────────────────────────────────────────┘

┌─ 비교 + 유사 ETF ────────────────────────┐
│  동종 보수 비교 바 + "이거 본 사람들이 본"   │
└──────────────────────────────────────────┘

┌─ 더 알아보기 ────────────────────────────┐
│  관련 질문 + 칼럼 + 운용보고서              │
└──────────────────────────────────────────┘
```

## 5. UI 통일 원칙 (Design System)

### 5-1. 페이지 골격 — 모든 페이지가 따른다

```tsx
<AppShell active="...">
  <PageContainer>              // 공통 max-width + padding
    <PageHero ... />            // 페이지 타이틀 (있어도 없어도)
    <Section title="...">       // 시멘틱 섹션
      <DataCard>...</DataCard>  // 통일된 카드
    </Section>
  </PageContainer>
</AppShell>
```

### 5-2. 컬러·여백 토큰만 사용

- `var(--rw-*)` 외에 hex/rgb 직접 쓰지 않기
- `var(--space-*)` 외에 px 직접 쓰지 않기 (icon, border 제외)
- `var(--type-*)` 외에 font-size 직접 쓰지 않기

### 5-3. 텍스트 톤 4단계

- `--type-display` — 페이지 타이틀
- `--type-title` — 섹션 타이틀
- `--type-body` — 본문 + 중요 숫자
- `--type-caption` — 라벨·메타

### 5-4. 데이터 표시 패턴

| 패턴 | 사용처 | 컴포넌트 |
|---|---|---|
| 큰 숫자 + 변화 | 현재가, 손익 | `<Stat>` (신규) |
| 라벨 + 값 + 톤 | 정보 그리드 | `<DataCell>` (신규) |
| Bar + 라벨 | 동종 비교 | `<CompareBar>` (있음) |
| Dot 1~5 | 위험 등급 | `<RiskMeter>` (있음) |
| Donut | 섹터 비중 | `<DonutChart>` (있음) |
| Mini bar | 분배금 등 | `<MiniBarChart>` (있음) |

### 5-5. 톤 매니페스토

- **숫자는 크고**, 라벨은 작고
- **친절한 표현** — "낮은 편이에요", "주의해보세요" (구어체)
- **단정짓지 않기** — "...일 수 있어요", "...편이에요"
- **이모지는 1개씩** — 🇺🇸 미국상장 같은 명확한 기능만
