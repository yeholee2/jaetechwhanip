# 🔴 v1 — 사용자 체류 직결 (먼저)

## v1-A. 마이페이지 (`/u/[id]` 또는 `/me`)

**현재:** 기본 프로필만
**목표:** a-ha "스파링 프로필" 같은 본인 활동 허브

**탭 구성:**
```
[프로필 헤더]
🦊 닉네임 [Lv.2]
가입 30일 · 답변 12 · 채택 3

[탭] 내 질문 / 내 답변 / 저장한 글 / 팔로우 / 활동
```

**각 탭:**
- 내 질문: 작성 질문 카드 리스트 (편집/삭제 가능, 본인만)
- 내 답변: 작성 답변 + 어느 질문에 달았는지 미리보기
- 저장한 글: Q&A·스파링·피드 모두 통합 (북마크 기능 신설)
- 팔로우: 팔로우한 토픽·작성자
- 활동: 최근 30일 참여 (투표/댓글/추천)

**스키마 추가:**
```sql
public.bookmarks
  - id, user_id, target_type ('question'|'sparring'|'column')
  - target_id, created_at
  - UNIQUE(user_id, target_type, target_id)

public.follows
  - id, user_id, target_type ('topic'|'user')
  - target_id, created_at
```

## v1-B. 토픽 페이지 6개 일괄 (`/topics/[slug]`)

**현재:** `/topics/finance-basics` 한 개만
**목표:** 6개 카테고리 모두 토픽 페이지 노출, 각 토픽 내 통합 콘텐츠

**페이지 구조:**
```
[토픽 헤더]
🏦 절세
세금 절감 노하우와 절세 상품 정보

[팔로우 +] [질문하기]

[탭] 질문 / 스파링 / 피드 / 인기

[해당 카테고리 콘텐츠 리스트]
```

**SSOT 매핑:**
- slug ↔ 카테고리: `docs/categories.md` + `docs/slug-spec.md` §1-2 매핑 그대로
- 6개 토픽 페이지를 `lib/categories.ts` 기반으로 자동 생성 (정적 라우트 또는 generateStaticParams)

## v1-C. 검색 결과 페이지 (`/search?q=...`)

**현재:** 미구현 추정
**목표:** 통합 검색 (Q&A·스파링·피드·토픽)

**구조:**
```
[검색바]  "ETF" 검색 결과

[탭] 전체 (147) / 질문 (89) / 스파링 (12) / 피드 (46) / 토픽 (3)

[결과 리스트]
- 카드 형식, 매칭 키워드 하이라이트
- 정렬: 관련도 / 최신
```

**구현:**
- Supabase Full Text Search (`tsvector`) — 한글 형태소 분석은 `pg_trgm` + 정규식 토크나이저
- 또는 단순 `ILIKE %q%` (초기엔 충분)
- 향후 정확도 필요시 Algolia/Meilisearch 도입

# 🟡 v2 — 완성도/엣지 (v1 후)

## v2-A. 미션 페이지 (`/missions`)

- 시즌제 미션 리스트 (현재 진행 중 + 다가오는)
- 미션 카드: 제목, 보상, 진행률, 마감일
- 클릭 → 상세 + 참여 버튼
- 마이페이지 연계 (내 미션 진행률)

## v2-B. 알림 페이지 (`/notifications`)

- 답변 달림 / 채택됨 / 팔로우한 토픽 새 글 / 신고 결과
- 읽음/안읽음 구분
- 푸시 알림 연동 (모바일 추후)

**스키마:**
```sql
public.notifications
  - id, user_id, type, payload (jsonb), read_at, created_at
```

## v2-C. 404 / 500 / 에러 페이지

- 친근한 일러스트 + "재테크한입에서 길을 잃으셨나요?" 카피
- 홈으로 이동 / 검색하기 CTA
- 500은 자동 슬랙 알림

## v2-D. 약관 / 개인정보처리방침 / 운영정책

- `/terms`, `/privacy`, `/policy` 정적 페이지
- 법무 검토 필요 (예호님 결정)
- footer에서 접근

## v2-E. 소개 페이지 (`/about`)

- "재테크한입은요?"
- 미션, 가치, 운영 원칙
- 팀/연락처

## v2-F. 작성자 페이지 (`/u/[id]` 외부 뷰)

- 본인 외 다른 사용자 프로필 보기
- 그 사람 질문/답변/팔로우 통계
- "팔로우하기" 버튼
