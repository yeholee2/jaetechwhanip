## ⚠️ 중요: 우리가 잘못 알고 있던 것들

### 1. **a-ha의 primary 색은 파란색이다 (녹색 X)**
- 추출본: `--color-primary: #0649db` (blue-700)
- 핵심 CTA bg: `#1e59da` (blue-600) — "나도 질문하기" 버튼 실제 값
- aha 브랜드 블루: `--color-blue-ahaBlue: #1c34de`
- → **우리는 #03C75A (네이버 그린) 유지가 맞음.** a-ha 그대로 따라가지 말 것. 브랜드 차별화.

### 2. **a-ha 카드는 사실 카드가 아니다**
- Q&A 피드의 "카드"는 background, border, box-shadow 모두 **0**
- 내용 블록을 **whitespace + 얇은 회색 디바이더**로만 구분
- 우리가 "카드 border 제거 + shadow만"이라고 했는데, **a-ha는 shadow도 없음**
- → 우리 ui-principles v2.1의 "shadow 단일"도 한 번 더 검토 필요 (옵션: 완전 borderless)

### 3. **카드 제목은 18px / 700 / line-height 18px** (예상보다 작고 타이트)

### 4. **스파링 페이지가 우리 생각과 완전히 다르다**
- 상단 2개는 **풀일러스트 배너 카드** (복싱링·키보드 일러스트, 1080x880 정도)
- 진행중 스파링만 위에 노출, 지난 스파링은 단순 텍스트 카드 리스트
- **댓글은 두 컬럼 분리 X** — 단일 피드인데 각 댓글에 사이드 뱃지

### 5. **스파링 투표 시스템이 정교하다**
- 투표 안 하면 댓글 작성 lock
- 라운드 번호 + 카운트다운 + 운영자 메모
- **성별 통계 도넛 차트** (남성/여성 분포)
- 댓글 정렬 옵션: "투표 항목별 ▼" + "내가 남긴 댓글" 필터

**Mint-green:** 03b26c, 02a262... — 우리 토큰과 비슷한 그린 계열. 우리 #03C75A를 mint-green-500 자리에 둘 수 있음.

### 1-5. 모션

**Q&A "카드" (실은 카드가 아님):**
- bg: transparent, border: 0, box-shadow: none
- padding: 24px (위/아래) 0 (좌우)
- 카드 사이: 1px 회색 디바이더 (`#e5e8eb` 또는 비슷)
- 프로필 이미지: 원형 56px
- 카테고리: 16px / 700 / black
- 보조 카테고리(작은): 16px / 400 / gray
- 닉네임 (link): blue / 600 / underline 없음
- 시간: 16px / 400 / gray-400
- 제목 (h2): 18px / 700 / gray-900 / line-height: 18px (타이트!)
- 발췌 (p): 16px / 400 / line-height: 24px / gray-700
- 답변수: 16px / 700 / gray-900 (예: "10명의 전문가가 답변했어요" — 숫자는 굵게)
- 액션 아이콘: 좋아요/싫어요/댓글 — 회색 outline 아이콘만

**필터 탭:**
- 인기/관심/답변 큰 텍스트 (24px+ / 700)
- 선택: gray-900, 비선택: gray-400

## 3. 우리(재테크한입) 적용 가이드

### 3-1. 채택할 것 (a-ha 그대로)
- ✅ Pretendard Variable 폰트 스택 (정확한 fallback 체인)
- ✅ Gray 스케일 11단계 (25-900) — `--gray-900: #191f28`을 본문 색으로
- ✅ 카드 = 무 background, 무 border, 무 shadow + whitespace + 얇은 디바이더 (border-bottom 1px gray-100)
- ✅ 모션: 150ms cubic-bezier(.4, 0, .2, 1)
- ✅ 카드 제목 18px / 700 / line-height 1.0
- ✅ 본문 16px / 400 / line-height 1.5
- ✅ 메타 14px / 400 / gray-600
- ✅ 정렬 토글 "기본순 ↑↓" 패턴
- ✅ 카테고리 칩 가로 스크롤 패턴
- ✅ 스파링 댓글: 단일 피드 + 사이드 뱃지 + "투표 항목별 ▼" 필터 (두 컬럼 분리 X)
- ✅ 스파링 투표: 투표 후 댓글 작성 가능 (lock)
- ✅ 라운드 번호 시스템

### 3-2. 채택 안 할 것 (브랜드 차별화)
- ❌ Primary 파란색 (#0649db) → 우리는 #03C75A (mint-green) 유지
- ❌ 14개 카테고리 → 우리는 6개 (`docs/categories.md`)
- ❌ 우측 영구 사이드바 → 우리는 단일 컬럼 720px (선택적으로 사이드바 추가 검토)
- ❌ a-ha 일러스트 (복싱링/키보드/돈) → 우리는 자체 일러스트 또는 도형/이모지

### 3-3. 결정 필요 (예호님)
- **🟡 카드 처리:** a-ha처럼 완전 무border/shadow + 디바이더만 갈래? 우리 ui-principles v2.1은 shadow만 (border 없음)
  - 옵션 A: a-ha 그대로 — shadow도 없음, 디바이더만
  - 옵션 B: ui-principles v2.1 유지 — 작은 shadow 있음
- **🟡 잉크 카드 썸네일:** a-ha는 좌측 정사각, 우리 spec은 위 풀너비 16:9
  - 옵션 A: 좌측 정사각 (a-ha 패턴, 더 컴팩트, 더 많은 카드 노출)
  - 옵션 B: 위 풀너비 16:9 (시각적 임팩트 큼, 카드 큼)
- **🟡 스파링 댓글 두 컬럼 vs 단일 피드:**
  - 옵션 A: a-ha처럼 단일 피드 + 사이드 뱃지 (구현 단순, 정렬 필터로 그룹핑)
  - 옵션 B: 두 컬럼 분리 (시각 임팩트, 구현 복잡)
